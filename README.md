# 정렬 알고리즘 시각화

이 탐구에서 구현한 정렬 알고리즘 시각화 프로그램을 통해 버블 정렬, 선택 정렬, 삽입 정렬의 동작 과정을 비교 및 관찰하였다

각 알고리즘의 결과는 다음과 같았다

버블 정렬
--
인접한 두 값을 반복적으로 비교하며 교환하는 과정이 매우 많이 발생함
애니메이션에서 두 막대가 계속 빨간색으로 바뀌며 비교되는 장면 자주 나타남
제일 오래 걸림

선택 정렬
--
비교가 많이 일어나지만 교환은 각 단계에서 1번만 발생함
막대를 보면 최소값이 선택되어 한 번에 이동하는 패턴이 보임
비교 횟수는 많지만 교환은 적음

삽입 정렬
--
배열 앞부분을 이미 정렬된 구간으로 보고 새로운 값을 적절한 위치에 삽입하는 과정이 보임
이미 정렬에 가까운 배열에서는 매우 빠르게 끝나는 모습을 확인
값이 이동할 때 막대들이 연속적으로 한 칸씩 이동하는 특징이 드러남

결과 분석
--
버블 정렬 : 비교, 교환 둘 다 많음 -> 가장 느림
선택 정렬 : 비교는 많지만 교환이 적음
삽입 정렬 : 정렬된 배열일수록 매우 빠름
